USER (11:24 AM):
why em and not px?

CLAUDE (11:24 AM):
Great question! Both work, but there are some practical differences:

`em` vs `px` for letter-spacing

`px` (pixels) - Absolute unit
* Fixed size regardless of font size
* More predictable and easier to visualize
* Example: `letter-spacing: -0.5px;`

`em` (relative to font size) - Relative unit
* Scales proportionally with font size
* If font is 16px, `1em` = 16px, so `0.01em` = 0.16px
* Maintains proportions if you change font sizes later

For your case, I'd actually recommend `px`:

```css
body {
    font-family: 'Times New Roman', Times, serif;
    background-color: #F4F4F4;
    letter-spacing: -0.3px; /* More intuitive */
}

.petition-body {
    flex: 2;
    text-align: justify;
    line-height: 1.8;
    font-size: 15px;
    letter-spacing: -0.2px; /* Easier to fine-tune */
}
```

Why I initially suggested `em`: Just habit from responsive design practices, but honestly for something as subtle as letter-spacing adjustments, pixels are totally fine and often easier to work with.

Try starting with `-0.2px` or `-0.3px` and adjust from there. You'll have much more intuitive control over the tightness!
